## 0:00–0:20｜钩子

- **预计时段 / 秒数：** 0:00–0:20，共 20 秒。
- **语气：** 前一句像替观众说出痛点；“直接到手”落重音，后半段加快一点。
- **画面 / 录屏动作：** 先快切“暂停—查词—切笔记”三个动作；说到“甩个链接”时切到 agent 对话框；末尾定格真实成品三联画。屏幕角落持续标注“演示素材：NASA Goddard；不代表背书”。
- **口播文本：**

<!-- SPOKEN:START hook -->
看英文教程视频，暂停查词十次就看不下去了？我做了个工具。甩个链接给 AI，双语字幕和学习笔记直接到手。不用在播放器、翻译器和笔记之间反复横跳。今天就用真实素材，把整条流程跑给你看。不用再折腾。
<!-- SPOKEN:END hook -->

## 0:20–1:00｜是什么：六个工具一个包

- **预计时段 / 秒数：** 0:20–1:00，共 40 秒。
- **语气：** 像报菜名，短促、清楚；每个工具名稍作停顿，不抢拍。
- **画面 / 录屏动作：** 打开 README 六工具表。每报一个名字，高亮对应一行；右侧同步出现“输入 → 产物”的极简图标。
- **口播文本：**

<!-- SPOKEN:START overview -->
它叫树树工具箱。六个工具装在一个包里。不必再学一个新应用。第一个，下载素材，保存公开视频或音频。第二个，获取字幕，优先用来源字幕。没有时，问过你再本地转写。第三个，翻译字幕，分块续跑。第四个，视频嵌字幕，可开关，也能永久显示。第五个，生成学习笔记，整理成复习稿。第六个，精读网页与 PDF，可全文翻译，也可做双语摘要。所有产物放进同一目录，可复跑。简单说，脚本做机械活，当前会话负责语言活。
<!-- SPOKEN:END overview -->

## 1:00–2:10｜实演：一句话到双语字幕和笔记

- **预计时段 / 秒数：** 1:00–2:10，共 70 秒。
- **语气：** 边操作边解释，像带朋友过一遍；数字读慢一点，验收结果用肯定语气。
- **画面 / 录屏动作：**
  1. 1:00 在新会话贴真实链接，输入“我要学这个”。
  2. 1:09 展示同一素材目录建立；终端快放字幕抓取、校验、分块与逐行翻译。
  3. 1:24 打开 `subs.bi.srt`，对照同一条 cue 的英文、中文与时间码；再切实际双语字幕画面。
  4. 1:35 打开 `notes.md`，扫过一句话核心、适合谁看、核心要点与概念。
  5. 1:43 **镜头在“常见误区”停 3 秒**；口播同步读出如实占位说明。
  6. 1:51 **镜头在“学以致用”停 4 秒**；口播同步解释具体行动。
  7. 1:58 回到成品目录，框选双语 SRT、笔记和软字幕视频。
- **口播文本：**

<!-- SPOKEN:START demo -->
现在实演。我贴上 NASA Goddard 的公开视频链接，只说：我要学这个。它长六十五点三二秒。先建素材目录。

接着抓来源英文字幕，清掉空白条目。再校验时间轴。结果是十九条。脚本拆成翻译块，当前会话逐行翻译。一条原文对应一行译文。行数错了，就报出问题块。

全部对齐后，合并成双语 SRT。每条开始和结束时间都没变。英文在上，中文在下。翻译不改时间码，也不漏行。

再看学习笔记。它不是把字幕缩短一遍。先给一句话核心，再写适合谁看、核心要点和概念。

素材没谈误区，它就如实写：素材未纠正常见误区。这里停三秒，别让 AI 为了完整乱补。

继续往下。学以致用给的是今天能做的动作。这里停四秒，不说空泛的以后试试。

最后打开目录。字幕、笔记、视频都在一起。想收藏，还能软封装进视频，字幕随时开关。全量回归测试守着流程。
<!-- SPOKEN:END demo -->

<!-- SPOKEN:START differences -->
它有三个差异。第一，零 API key。翻译和总结，就是当前会话。下载、切块和校验都由本地脚本复跑。每一步都有产物。第二，代码开源。macOS、Windows 和 Linux 都能跑。第三，目标是十五分钟装好。全新三点十一环境实测，安装和自检约五十九秒。版本不兼容时，工具会给出换版本指引。其他能力照常用，出错也说人话。
<!-- SPOKEN:END differences -->

## 计时校准（不口播）

- 统计口径：只统计五组 `SPOKEN:START` 与 `SPOKEN:END` 标记之间的 CJK 统一表意文字；标题、标点、英文、数字和舞台说明均不计入。
- 分段 CJK 字符与 250 字 / 分钟换算：钩子 83（19.92 秒）；六工具 161（38.64 秒）；实演 283（67.92 秒）；差异点 112（26.88 秒）；行动号召 74（17.76 秒）。
- 口播 CJK 总数：**713**；所有按句号、问号、叹号、冒号或分号切分的句子均不超过 20 个 CJK 字符。
- 按每分钟 250 个 CJK 字符上限换算：**171.12 秒，即 2:51.12**，位于 2:50–3:10 目标区间；五段分别落在 20 / 40 / 70 / 30 / 20 秒窗口内。
- 离线 TTS 分段结果：五段统一使用 macOS `say` 的 `Tingting` 中文声音与 `-r 215`，`afinfo` 实测依次为 **18.58 / 39.79 / 67.89 / 29.86 / 19.63 秒**，没有给超时段单独加速。
- 同一设置合成完整口播，`afinfo` 实测 **176.66 秒，即 2:56.66**；分段文件因各自收尾，时长合计为 175.75 秒。所有合成文件只在 `/private/tmp`，未进入仓库。
- 实演七段的 TTS 累计位置用于校准动作：贴链接 1:00、抓取与翻译约 1:09、双语 SRT 约 1:24、笔记约 1:35、常见误区约 1:43、学以致用约 1:51、成品目录约 1:58；常见误区与学以致用的停留分别和对应口播同步 3 秒、4 秒。
- `say` 的速率刻度不等于 CJK 字符 / 分钟，TTS 也不是人工朗读；以上结果只作节奏辅助。正式录制仍以分段时间轴为准，并在同一语速下人工校准一次。
