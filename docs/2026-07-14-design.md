把"看英文视频/资料学习"的整套日常流程——下载、字幕、翻译、字幕入视频、精华笔记、资料总结——整合成一个 agent skill,自己天天用,也让 GitHub 上的陌生人 15 分钟内跑起来。

## 分工总原则

**脚本干机械活,Claude(agent)干语言活。**

- 机械活:下载、抓字幕、语音转写、SRT 排版合并、ffmpeg 封装 → Python 脚本,行为确定、可测试
- 语言活:翻译、写笔记、摘要 → 会话内的 agent 直接做,不调外部 API,不需要任何 key

## 六个工具的行为定义

| 工具 | 输入 | 处理 | 输出 |
|------|------|------|------|
| 下载视频 | 视频链接 | `fetch.py`(yt-dlp),按 `video_quality` 限清晰度 | 视频文件 + `meta.json` |
| 获取字幕 | 链接或本地音视频 | 先抓官方/自动字幕;没有则提示后用 `transcribe.py` 转写 | 原文 SRT |
| 翻译字幕 | 原文 SRT | agent 分批翻译;`srt_tools.py` 管分块、双语排版、时间轴校验、断点续跑 | 双语 SRT |
| 字幕入视频 | 视频 + 双语 SRT | `mux.py` 默认软字幕封装;可选硬烧录(慢,明示耗时) | 带字幕视频 |
| 双语精华笔记 | 字幕/文字稿/文章 | agent 按内置模板写:tldr / 核心概念 / 常见误区 / 学以致用(树树 v2 口径移植) | Markdown 笔记 |
| 资料翻译总结 | 网页 URL / PDF / 文本 | agent 读取后按需全文翻译或双语摘要 | Markdown |

### 预设流水线(SKILL.md 定义,一句话触发)

- **学习(默认)**:"我要学这个视频" → 获取字幕 → 翻译字幕 → 精华笔记
- **收藏**:学习流水线 + 下载视频 + 字幕入视频(离线全家桶)
- 每个工具均可单独点名使用

每个素材一个文件夹:


## 配置

`config.example.json` → 首次使用引导生成 `config.json`(gitignore):

```jsonc
{
  "output_dir": "~/ShushuStudy",
  "native_lang": "zh",          // 母语,译文和笔记用它写;改一个字段即可服务任何母语用户
  "source_lang": "auto",
  "whisper_model": "small",     // tiny/small/medium
  "video_quality": "1080",
  "subtitle_layout": "original-top"
}
```

## 跨平台保障(无实测机器的三层保险)

1. **依赖尽量走 pip**:yt-dlp、faster-whisper 均 `pip install`,三平台行为一致
2. **ffmpeg 双保险**:优先系统 ffmpeg,检测不到自动退回 pip 静态版(`static-ffmpeg`)
3. **`doctor.py` 自检**:检查 Python 版本/依赖/ffmpeg/输出目录可写,缺项打印当前平台对应安装命令(brew/winget/apt)

**代码纪律**:全程 `pathlib`;文件读写显式 UTF-8(防 Windows GBK);文件名清洗 Windows 非法字符;不用 `shell=True`。

**CI**:GitHub Actions 矩阵(macOS/Windows/Ubuntu),每次提交跑:依赖安装 → doctor → srt_tools 单元测试 → ffmpeg 生成 1 秒测试视频封装字幕的冒烟测试。不联网测 YouTube(不稳定 + ToS)。

## 错误处理原则

- **人话报错**:报"是什么 + 怎么办",不甩堆栈;地区锁/会员内容/无字幕各有明确提示
- **长任务可续跑**:翻译按块进行,中断重跑只补缺块;转写前预报耗时
- **不静默降级**:官方字幕缺失需转写时,先告知耗时并确认,不闷头跑

## 文档与复用体验

README(中文为主 + English quick start):一句话介绍与工具清单 → 三步安装(clone → doctor → 配置+软链接)→ 各 agent 接入方法(Claude Code / Codex / 其它)→ 每工具一条用法示例 → FAQ → 合规声明(下载仅供个人学习,遵守各平台 ToS)。

**验收标准:一台全新电脑,照 README 从 clone 到出第一份双语笔记 ≤ 15 分钟,全程不需要任何 API key。**

## 不做什么(YAGNI)

- 不做 GUI / 菜单栏 app(被本项目取代)
- 不调外部翻译/LLM API,不管理任何 key
- 不移植 n-gram 原创性闸门
- 不做批量队列/定时任务(先满足"甩一个链接"的单件流)
- 不在 CI 里联网测下载
